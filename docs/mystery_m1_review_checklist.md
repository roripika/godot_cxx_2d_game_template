# Mystery M1 レビュー観点チェックリスト

## 使い方
- M1 PR レビュー時に、本チェックを上から順に確認する。
- `NG` がある場合は、理由と修正案を PR コメントに残す。

## A. 入口統一
- [ ] `project.godot` の `run/main_scene` が意図した導線を向いている
- [ ] Mystery 起動時に `res://samples/mystery/karakuri_mystery_shell.tscn` が使用される
- [ ] 実行導線が複数残っていない、または残す理由が明記されている

## B. 旧導線整理
- [ ] `samples/mystery/main_mystery.tscn` など旧導線の扱いが明確
- [ ] 旧導線に依存する参照が残っていない（残る場合は意図が記載）
- [ ] 削除した場合、参照切れやロードエラーが発生しない

## C. 動作確認
- [ ] `./dev.sh run mystery` で正常起動
- [ ] 起動後の初期画面から進行不能が発生しない
- [ ] 起動フロー変更による既存機能の退行がない

## D. ドキュメント整合
- [ ] `README.md` の起動手順が実装と一致
- [ ] `docs/mystery_design.md` の導線説明が実装と一致
- [ ] `TASK.md` の M1 チェック項目が更新されている

## E. PR 品質
- [ ] PR に `Scope / Out of Scope / Test / Risks` が記載されている
- [ ] 変更対象が M1 範囲に収まっている
- [ ] 持ち越し課題が次のマイルストーンに明示されている

## F. マージ判定
- [ ] A〜E がすべて `OK`
- [ ] 重大な未解決リスクがない
- [ ] マージ後の次作業（M2開始条件）が明確
