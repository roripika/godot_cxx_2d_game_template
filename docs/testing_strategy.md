# テスト戦略 / Mystery Demo Test Strategy

## 概要

Mystery デモには 3 種類のテストが存在する。  
AI が生成・変更したコードは必ず **該当ティアのテストをすべて通過してからマージ**する。

```
変更規模 ─────────────────────────────────────────────────────────►
小 (GDScript のみ)         中 (C++ or YAML)         大 (アーキテクチャ変更)
     │                         │                              │
  smoke                    smoke + e2e               smoke + e2e + monkey
```

---

## テスト一覧

### Tier 1 : smoke （軽量起動確認）

| 項目 | 内容 |
|---|---|
| **ファイル** | `samples/mystery/scripts/karakuri_smoke.gd` |
| **実行コマンド** | `./dev.sh smoke` |
| **所要時間** | 約 5 〜 10 秒 |
| **対象変更** | 全ての変更（最低限）|

**検証内容（4点）:**

1. `karakuri_mystery_shell.tscn` が正常にロードされる
2. 必須ノードが存在する  
   `ScenarioRunner` / `InteractionManager` / `DialogueUI` / `InventoryUI` / `SystemUiLayer`
3. プロローグダイアログキー `mystery.prologue.system` が表示される
4. EN/JA ロケール切替でテキストが変わる

**検証しないこと:**

- シーン遷移・ゲームフロー全体

---

### Tier 2 : e2e （全シナリオ通しテスト）

| 項目 | 内容 |
|---|---|
| **ファイル** | `samples/mystery/scripts/karakuri_scenario_smoke.gd` |
| **実行コマンド** | `./dev.sh e2e` |
| **所要時間** | 約 60 秒 |
| **対象変更** | C++ 変更 / YAML 変更 / シーン変更 |

**検証内容:**

| フェーズ | 検証項目 |
|---|---|
| プロローグ | ダイアログが表示される、ロケール翻訳が異なる |
| 倉庫（調査モード） | NPCが非表示、証拠ホットスポットが反応する |
| 倉庫（会話モード） | NPC が表示される、田中・佐藤のダイアログが起動する |
| モード切替 | ボタンテキストが切り替わる、表示ノードが切り替わる |
| インベントリ | 開閉動作、CloseButton で閉じられる |
| インベントリ中 NPC | インベントリ表示中のクリックが無視される |
| 推理 | 間違い選択でHP減少、正解選択で対決へ遷移 |
| 対決 | 証拠提示の正誤判定、HP 管理 |
| グッドエンド | 正解証拠連続提示でエンディングへ |
| バッドエンド | HP 枯渇でエンディングへ |

---

### Tier 3 : monkey （ランダム操作耐久テスト）

| 項目 | 内容 |
|---|---|
| **ファイル** | `samples/mystery/scripts/karakuri_monkey.gd` |
| **実行コマンド** | `./dev.sh monkey` |
| **所要時間** | 約 30 〜 60 秒（デフォルト 400 アクション） |
| **対象変更** | C++ 変更 / 入力処理変更 / UI 状態管理変更 |

**目的:**  
AI が実装した機能が、想定外の操作順序・タイミングに対しても安全であることを確認する。  
スクリプトテストでは発見できない**レースコンディション的なバグ**を検出する。

**ランダムアクション（比重あり）:**

| アクション | 比重 | 内容 |
|---|---|---|
| 画面ランダムクリック | 4/20 | 1280×720 内の任意座標 |
| ホットスポットクリック | 3/20 | 倉庫の既知ホットスポット座標をランダム選択 |
| モード切替ボタン | 2/20 | Investigate ↔ Talk の切替 |
| インベントリ開く | 3/20 | `show_inventory()` |
| インベントリ閉じる | 3/20 | CloseButton を押す |
| 会話スキップ | 5/20 | タイピングスキップ / dialogue_finished 発火 |

**実行後の不変条件チェック:**

1. `SceneContainer` が存在する（クラッシュしていない）
2. インベントリが開閉できる（デッドロックになっていない）
3. `ModeToggleButton` を押すとテキストが変わる（UI が壊れていない）

**環境変数によるカスタマイズ:**

```bash
# アクション数を変える（CI ではデフォルト 400 でよい）
MONKEY_ACTIONS=1000 ./dev.sh monkey

# 再現性のためシードを固定
MONKEY_SEED=12345 ./dev.sh monkey
```

> **再現時の手順:** 失敗したときは出力の `seed=XXXXXX` を控える。  
> `MONKEY_SEED=XXXXXX ./dev.sh monkey` で同じ操作列を再現できる。

## AI 実装ガイドライン

### 必須テストの対応表

| 変更種別 | smoke | e2e | monkey |
|---|:---:|:---:|:---:|
| GDScript (UI 表示のみ) | ✅ 必須 | 推奨 | — |
| GDScript (入力ハンドラ変更) | ✅ 必須 | ✅ 必須 | ✅ 必須 |
| C++ クラス変更 | ✅ 必須 | ✅ 必須 | ✅ 必須 |
| YAML シナリオ変更 | ✅ 必須 | ✅ 必須 | — |
| 新ホットスポット追加 | ✅ 必須 | ✅ 必須 | ✅ 必須 |
| UI ノード追加/削除 | ✅ 必須 | ✅ 必須 | — |

### ホットスポット追加時の更新ルール

新しいホットスポット/NPC を倉庫に追加した場合:

1. `karakuri_monkey.gd` の `HS_POSITIONS` 配列に座標を追加する
2. `karakuri_scenario_smoke.gd` にインタラクションのアサーションを追加する

### CI 推奨設定

```bash
# 最小 CI (毎コミット)
./dev.sh smoke

# フル CI (PR マージ前)
./dev.sh smoke && ./dev.sh e2e && ./dev.sh monkey
```

## 各テストの設計思想

```
smoke   ─ 「動くか？」    起動の最低限保証。5 秒で判断。
  │
  └── e2e ─ 「正しく動くか？」  正解/失敗の全ルートを設計通りに通過。
              │
              └── monkey ─ 「壊れないか？」  想定外操作でも安全に動作。
```

AI による実装では「正しく動く」だけでなく「どんな順序で操作されても壊れない」ことが  
ゲーム品質には必須であり、monkey テストがその保証を担う。
