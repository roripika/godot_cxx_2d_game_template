# Human Gate Asset Policy

**ステータス**: draft v0.1  
**作成日**: 2026-04-03  
**目的**: 各アセット生成ラインに対して、AG-1〜AG-4 の判定観点、PASS / REWORK / STOP、主軸 repo に残す証跡を定義する。

---

## 1. 目的

本ドキュメントは、UI / 音 / ボイス / キャラ画像 / モーション / エフェクト / 動画など、
アセット種別ごとに Human Gate の観点差を整理し、主軸 repo 側で一貫した判断ができるようにする。

目的は以下。

1. アセット種別ごとに異なる review 観点を明文化する
2. PASS / REWORK / STOP の判断基準を揃える
3. 各ライン repo での詳細判断を、主軸 repo 側で追跡可能にする
4. 「生成したが採用しない」「再生成が必要」「統合できる」を混同しないようにする

---

## 2. 適用範囲

本 policy は以下に適用する。

- UI asset pipeline
- audio asset pipeline（SFX / BGM / voice）
- character image pipeline
- motion pipeline
- effect pipeline
- video pipeline

scenario generator 本体の HG とは別系統だが、Gate の思想は整合させる。

---

## 3. 共通 Gate 定義

| Gate | 目的 | 共通観点 |
|:---|:---|:---|
| AG-1 | 生成前確認 | 用途、入力仕様、世界観、利用先が妥当か |
| AG-2 | 生成直後レビュー | 明らかな破綻、形式不備、用途逸脱がないか |
| AG-3 | 品質確認 | 品質・一貫性・利用耐性があるか |
| AG-4 | 統合前確認 | 実ゲーム内で利用可能か、他資産と整合するか |

---

## 4. 判定値

各 Gate の判定は以下のいずれかとする。

- `PASS`
- `REWORK`
- `STOP`

### 4-1. 原則

- `PASS`: 次の Gate または統合工程へ進む
- `REWORK`: 修正または再生成し、同じ Gate へ戻る
- `STOP`: 現タスクでは続行せず、設計・要件・責務分離の見直しへ戻す

---

## 5. アセット種別ごとの Gate 観点

### 5-1. UI

#### AG-1

- 使用画面が明確か
- ボタン / パネル / バナーなどの用途が定義されているか
- 解像度やレイアウト前提が明確か

#### AG-2

- 文字が読めない、余白が破綻している、押下領域が不自然などの明確な崩れがないか
- 世界観から明らかに逸脱していないか

#### AG-3

- 視認性
- 可読性
- 押下対象として自然か
- 同一画面内での一貫性

#### AG-4

- 実際の画面に置いて使えるか
- 他 UI と並べて違和感がないか

---

### 5-2. SFX

#### AG-1

- 用途（UI / hit / notice など）が定義されているか
- 再生タイミングが想定されているか

#### AG-2

- ノイズ、無音過多、極端な音量差がないか

#### AG-3

- 用途に合った長さか
- 連打再生に耐えるか
- 聞き疲れしないか

#### AG-4

- 実ゲーム上の再生位置で違和感がないか
- 他の SFX と衝突しないか

---

### 5-3. BGM

#### AG-1

- 使用シーン（title / field / battle など）が明確か
- ループ前提か単発前提かが定義されているか

#### AG-2

- 明らかな破綻、ノイズ、無音、音量異常がないか

#### AG-3

- 世界観に合うか
- ループ品質に問題がないか
- 長時間再生に耐えるか

#### AG-4

- シーンに乗せたときに違和感がないか
- 他音声とのバランスが取れるか

---

### 5-4. Voice

#### AG-1

- 話者、感情、台詞用途が明確か
- 台本との対応が取れているか

#### AG-2

- 明瞭性があるか
- 明らかな読み違い、破綻、ノイズがないか

#### AG-3

- 話者一貫性があるか
- 感情表現が意図と合うか
- 同一キャラ内で違和感がないか

#### AG-4

- 字幕や台本と整合するか
- 実ゲーム上の再生長やテンポに合うか

---

### 5-5. Character Image

#### AG-1

- キャラID、用途（立ち絵 / カード / 差分）が定義されているか
- 世界観や衣装前提が明確か

#### AG-2

- 明らかな破綻、崩れ、余分なパーツ、構図不整合がないか

#### AG-3

- キャラ一貫性
- 差分整合
- 量産耐性
- 他画像との統一感

#### AG-4

- 実ゲームでの表示サイズに耐えるか
- 他資産と並べて違和感がないか

---

### 5-6. Motion

#### AG-1

- 対象キャラ / rig / clip 用途が定義されているか
- idle / attack / hit などの目的が明確か

#### AG-2

- 明らかな破綻、接続不能、命名崩れがないか

#### AG-3

- ループ自然さ
- 接続自然さ
- 再利用性
- 他 clip との一貫性

#### AG-4

- 実機で違和感なく使えるか
- 他アニメや演出と接続可能か

---

### 5-7. FX

#### AG-1

- 用途（hit / slash / aura / flash）が明確か
- 表示レイヤや発火タイミングが想定されているか

#### AG-2

- 明らかな破綻、ノイズ、視認不能がないか

#### AG-3

- 過剰に見づらくないか
- 尺が長すぎないか
- 用途に対して十分な視認性があるか

#### AG-4

- 実ゲーム画面で過不足なく機能するか
- 他 UI / motion / BGM と干渉しないか

---

### 5-8. Movie

#### AG-1

- 用途（OP / tutorial / ad / cutscene）が明確か
- 想定尺、解像度、音声有無が明確か

#### AG-2

- 明らかな崩れ、破綻、フレーム飛び、ノイズがないか

#### AG-3

- 尺、構成、見やすさ、同期品質
- 用途に対する完成度

#### AG-4

- 実際の再生環境で成立するか
- 導線やテンポを阻害しないか

---

## 6. PASS / REWORK / STOP の目安

| 判定 | 典型例 |
|:---|:---|
| PASS | 用途に適合し、次工程へ進めてよい |
| REWORK | 方向性は合っているが、品質・差分・形式の修正が必要 |
| STOP | 要件不明、用途不整合、責務外、世界観衝突などでこのまま進めるべきでない |

---

## 7. 主軸 repo に残す証跡

各 asset task について、主軸 repo 側には最低限以下を残す。

- asset type
- task 名
- Gate ごとの判定
- 最終 status
- 承認者またはレビュー者
- `REWORK` / `STOP` の理由
- 統合先または未統合理由

---

## 8. 主軸 repo に残さないもの

- 各ライン内部の細かい生成ログ全文
- モデル固有のパラメータ実験履歴
- 再生成の全試行ログ
- 専用 repo 内だけで完結する詳細比較画像・音声
