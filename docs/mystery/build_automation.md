# Godot Distribution Build Automation (CLI)

## 概要

本プロジェクトにおいて、複数存在するデモシーン（ミステリーデモ、メインメニューなど）を開発者の手作業（GUI操作）なしで独立したアプリケーションとして配布（パッケージング）するための自動化スクリプト `build_demo.sh` を導入しました。本ドキュメントでは、その技術選定の理由と要件定義についてまとめます。

## 要件（Requirements）

1. **コマンドプロンプト（CLI）からのワンタッチ実行**
   - CI/CDパイプライン（GitHub Actionsなど）での将来的な利用を見据え、GUIを開かずに完全自動でビルド可能であること。
2. **対象デモ（メインシーン）の動的切り替え**
   - ひとつのGodotプロジェクト内に複数の独立したゲームデモや検証用シーンが存在するため、引数によって「どのデモを起動するアプリとして出力するか」を指定できること。
3. **マルチプラットフォーム出力**
   - 開発用の macOS (`.app`) だけでなく、Windows (`.exe`)、Android (`.apk`)、iOS (`.ipa`) の出力要件をカバーできる汎用性を持つこと。
4. **環境非依存（可能な限りのフォールバック）**
   - エクスポートテンプレートが不足している場合、適切なエラーメッセージにより開発者を誘導すること。

## 技術選定とアプローチ

### 1. `Godot CLI (--headless --export-release)` の採用
外部のサードパーティ製パイプラインツール（fastlaneなど）を使用せず、Godot公式が提供するヘッドレスコマンドラインインターフェースを直接利用するアプローチを採用しました。
- **理由**: 公式の機能であるため、Godotのバージョンアップに対する追従が容易であり、ビルド固有の依存関係（Node.jsやPythonパッケージなど）を増やすことなくシェルスクリプトのみで完結するためです。

### 2. `project.godot` の一時書き換えによるシーン切り替え
Godotのエクスポート機能には「コマンドライン引数で出力アプリのメインシーンを指定する」直接的なオプションが存在しません。そのため、スクリプト実行時に一時的に `project.godot` の `run/main_scene` のパスを `sed` コマンドで上書きし、エクスポート完了後に元に戻す手法を採用しました。
- **理由**: Godotの標準仕様を変えずに済むこと、またエクスポートのたびにGUIで `Project Settings` を開いて手作業でメインシーンを差し替える人的ミスを完全に排除できるためです。

### 3. `export_presets.cfg` の動的生成
複数プラットフォーム（Mac/Win/Android/iOS）向けの設定ファイル `export_presets.cfg` をリポジトリにコミットし、ビルド時にその設定（Preset名）を呼び出す形式を採用しました。ビルド対象のデモ名（Mystery, Main等）に応じて、出力されるバイナリ名（`application/name` や `export_path`）が適切に反映されるよう設計しています。

### 4. エクスポートテンプレートの管理方針（手動インストールへの誘導）
Godot公式の標準環境（`4.3.stable` など）であれば、スクリプト内からGithubリポジトリを叩き `wget/curl` でテンプレート(`.tpz`)を自動インストールすることが可能です。
しかし、本プロジェクトは **C++ GDExtension** を多用しており、独自のカスタマイズが施されたエンジン (`4.6.stable.official.89cea1439` 等) もしくは特定のコンパイルバージョンの使用が前提となっています。
- **判断**: 無理に公式の別バージョンテンプレートを自動ダウンロードすると、バイナリ不一致による即時クラッシュや後発的な互換性バグなどの深刻な事態を引き起こすため、**自動ダウンロード機能はあえてオミット**し、コンソール上で「手動でエディタから正しいテンプレートをインストールしてください」とエラーログでユーザーを誘導する安全な設計（Fail-fast）を選定しました。

## 想定される使用フロー

```bash
# 権限付与（初回のみ）
chmod +x build_demo.sh

# ミステリーデモを Windows (.exe) 向けに出力
./build_demo.sh mystery win

# メインメニューデモを Android (.apk) 向けに出力
./build_demo.sh main apk

# ミステリーデモを Mac (.app) 向けに出力
./build_demo.sh mystery mac

# ミステリーデモを iOS (.ipa) 向けに出力
./build_demo.sh mystery ipa
```

※ iOS / Androidビルド実行時は、Godotエディタ（Editor Settings）側で予め Android SDK や iOS Provisioning Profile のパス設定が完了している必要があります。
