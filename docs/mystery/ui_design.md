# 画面構成・UI設計 (UI & Screen Design)

## 1. 画面一覧 (Scenes)

### 事務所シーン (OfficeBase)
- **概要**: プロローグ、中休み、および推理パートの舞台。
- **UI要素**: ボスや助手との会話UI、推理選択ボタン。

### 倉庫シーン (WarehouseBase)
- **概要**: 調査および対決パートの舞台。
- **特殊仕様**: 「調査モード」と「会話モード」の動的切り替え。

## 2. UIコンポーネント設計（共通レイアウトポリシー準拠）

本プロジェクトは `docs/ui_layout_policy.md` に基づき、UIを4つの階層グループに分類し、同一グループ内での重なりを禁止しています。

### レイヤー構成 (Priority Groups)
1. **システム表示 (system / Layer 30)**: 常設情報。HPバー、言語切替、インベントリボタン。
2. **補助情報 (instant_subinfo / Layer 20)**: 短時間の通知、ヒント。インベントリ画面。
3. **メイン文脈 (main / Layer 10)**: 会話ウィンドウ、選択肢、証言システム。
4. **サブ情報 (sub / Layer 0)**: 最背面。

### 主要コンポーネント
- **ダイアログUI (DialogueUI)**: `main` レイヤー。話者名、本文、選択肢を表示。
- **インベントリUI (InventoryUI)**: `instant_subinfo` レイヤー。表示中はGDExtension層で背景の入力を遮断。
- **操作ボタン**: 
    - `InventoryButton`: `system` レイヤー（右上）。常時アクセス可能。
    - `ModeToggleButton`: 倉庫シーン固有。`main` レイヤー（右上、インベントリボタン下）。立ち絵や会話との重複を避ける配置。
- **システムヒント (HintLabel)**: `instant_subinfo` レイヤー。`system` レイヤーのHealthBar下に配置。

## 3. インタラクティブ演出 (Visual Feedback)

### ダイアログ演出
- **読み進めインジケーター (Proceed Indicator)**: メッセージ送りが可能な状態で点滅する下向きの三角形。`DialogueUI` の右下に配置。
- **選択肢の装飾**: 現在の選択肢に「▶」などの装飾を加え、視認性を向上。
- **立ち絵と背景の排他表示**: キャラクターの会話ポートレート（立ち絵）が表示されている間は、対応する背景のキャラクターグラフィックを非表示にする。これにより、同一キャラクターが画面上に二重に表示される違和感を防ぐ。(`portrait_shown` / `portrait_hidden` シグナルで制御)

### シナリオ連動エフェクト (Karakuri Actions)
`KarakuriScenarioRunner` から呼び出し可能な独自アクションとして実装。
- **画面フラッシュ (flash_screen)**: ダメージ時やショック時に画面全体を特定の色でフラッシュ。
- **画面シェイク (shake_screen)**: ボスの激昂などを表現するためにUI全体を振動。
- **実装レイヤー**: `DialogueUIAdvanced` クラスにアニメーションロジックを実装し、C++側からバインド。

## 4. 入力制御仕様 (Input Propagation)

| 制御レベル       | 実装詳細                                           | 目的                                             |
| ---------------- | -------------------------------------------------- | ------------------------------------------------ |
| **最前面(UI)**   | `_unhandled_input` を使用。                        | UIクリック時に背景へイベントを流さない。         |
| **GDExtension**  | `KarakuriScenarioRunner` 内でのフラグチェック。    | インベントリ表示中に背景の当たり判定を無効化。   |
| **配置レイヤー** | 立ち絵（PortraitRect）の `mouse_filter = IGNORE`。 | 立ち絵が背景オブジェクトのクリックを邪魔しない。 |
| **可視性**       | `is_visible_in_tree()` による判定。                | 隠れたホットスポットをクリック対象外とする。     |
