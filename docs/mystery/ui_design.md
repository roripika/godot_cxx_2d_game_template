# 画面構成・UI設計 (UI & Screen Design)

## 1. 画面一覧 (Scenes)

### 事務所シーン (OfficeBase)
- **概要**: プロローグ、中休み、および推理パートの舞台。
- **UI要素**: ボスや助手との会話UI、推理選択ボタン。

### 倉庫シーン (WarehouseBase)
- **概要**: 調査および対決パートの舞台。
- **特殊仕様**: 「調査モード」と「会話モード」の動的切り替え。

## 2. UIコンポーネント設計

### ダイアログUI (DialogueUI)
- **構成**: 名前欄、テキスト欄、およびポートレート表示エリア。
- **操作**: 左クリックでテキスト送り。
- **特殊機能**: テキスト演出（タイピング速度）、ポートレート遷移（スライド/ズーム）。

### インベントリUI (InventoryUI)
- **構成**: 証拠品スキャフォールド（グリッド形式）、詳細情報（画像・説明）。
- **制御**: アイテム入手時に自動ポップアップ。表示中は背景のクリックをC++層で遮断。

### ライフUI (HealthBar)
- **位置**: 画面左上。
- **構成**: ハートアイコンまたはゲージ形式。
- **制御**: `health` フラグの値と連動。

### システムヒント (HintLabel)
- **位置**: 画面左上（HealthBarの下）。
- **構成**: 縁取り付きのテキストラベル。
- **内容**: プレイヤーの現在の目的をシステム文言で提示。

## 3. 入力制御仕様 (Input Propagation)

| 制御レベル | 実装詳細 | 目的 |
|---|---|---|
| **最前面(UI)** | `_unhandled_input` を使用。 | UIクリック時に背景へイベントを流さない。 |
| **GDExtension** | `KarakuriScenarioRunner` 内でのフラグチェック。 | インベントリ表示中に背景の当たり判定を無効化。 |
| **配置レイヤー** | 立ち絵（PortraitRect）の `mouse_filter = IGNORE`。 | 立ち絵が背景オブジェクトのクリックを邪魔しない。 |
| **可視性** | `is_visible_in_tree()` による判定。 | 隠れたホットスポットをクリック対象外とする。 |
