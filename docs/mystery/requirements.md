# 要件定義 (Requirements Definition)

## 目的
Karakuri C++ SDK を使用して、2Dアドベンチャーゲーム（ミステリージャンル）の標準的な実装パターンを提示する。

## ターゲットユーザー
- Karakuri SDK を用いてアドベンチャーゲームを開発したいエンジニア。
- シナリオ駆動型ゲームのデータ構造とロジック分離の好例を知りたい開発者。

## 機能要件 (Functional Requirements)

### 1. シナリオ進行・実行
- YAML形式のシナリオ定義ファイルに基づき、会話、分岐、フラグ制御を順次実行できること。
- シナリオ実行中、プレイヤーの入力を適切に制御（ロック/アンロック）できること。

### 2. インタラクションシステム
- マウスクリックによる背景調査（Hotspot）機能。
- NPCとの会話および聞き込み機能。
- モード切替（調査モード ⇔ 会話モード）による情報の出し分け。

### 3. インベントリ・証拠品管理
- 入手したアイテム（証拠品）を一覧表示し、詳細（画像・説明文）を確認できること。
- 特定のアイテムを所持していることがシナリオ進行の条件（フラグ）になること。

### 4. 推理・対決システム
- 収集した証拠品を提示（突きつける）し、矛盾を暴くゲーム性の実装。
- HP（ライフ）概念の導入。推理ミスによりHPが減少し、マルチエンディングへ分岐すること。

### 5. マルチ言語対応
- 会話文、UIラベル、アイテム説明のすべてが、動的な言語切り替えに対応していること。

### 6. 自動検証 (Smoke Test)
- ヘッドレス環境で、ゲーム開始からエンディングまでの一連の流れを自動検証できること。
