詳解

Karakuri OS のメモリサンドボックス機構 (旧 FlagService)。 単一のグローバル辞書による状態汚染を防ぐため、名前空間（Namespace）と 生存期間（Scope）による厳格なメモリ管理・分離を行う。

内部データ構造

スコープごとに独立した Dictionary を持ち、その中で Namespace により ツリー化する: { namespace: { key: value } }

後方互換 (旧 FlagService)

  • set_flag / get_flag / has_flag を namespace="core", scope=SCOPE_GLOBAL で ラップするメソッドとして提供。
